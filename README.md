# Описание проекта
Создан планировщик задач, который хранит задачи. Каждая задача имеет свой дедлайн и заголовок с комментарием.
Задачи могут повторяться по заданному правилу: 
например, ежегодно, через какое-то количество дней, в определённые дни месяца или недели. 
Если отметить такую задачу как выполненную, она переносится на следующую дату в соответствии с правилом. 
Обычные задачи при выполнении будут просто удаляться.

# Список выполненных заданий со звездочкой
Выполнены все задания со звездочкой.

Реализованы такие переменные окружения: 

	`TODO_PORT` - берет указанный порт из переменного окружения и оспользует его для прослушивания.
	Если переменная окружения не задана, то используется порт по умолчанию :7540

	`TODO_DBFILE` - дает возможность определять путь к файлу БД через эту переменную окружения.
	Если он не задан используется файл с названием scheduler.db

	`TODO_PASSWORD` - берет указанный пароль из переменного окруженя для аутентификации. 
	Если пароль не задан, приложение работает без аутентификации.

Реализована возможность вычисления следующей даты задачи по месяцам и неделям.

Реализована возможность поиска задачи через поисковую строку.
Поисковая строка ищет задачу по названию задачи и по комментарию, также работает поиск по дате.

Реализована аутентификация и работа с докером.

# Инструкция по запуску кода локально
Сервер можно запустить с переменными окружения, так и без них.

Для запуска сервера без переменных окружения используется команда

go run .

Для запуска сервера с использованием переменных окружения нужно будет задать их напрямую в команде. например: 

```
TODO_PASSWORD=12345 TODO_DBFILE='example.db' TODO_PORT=:8080 go run .
```

В браузере для того, чтобы зайти на главную страницу, например с портом по умолчанию, используется адрес:

http://localhost:7540/

если пароль был задан, то вас перенаправят на страницу аутентификации

# Запуск тестов
Так как реализованы все основные и дополнительные задания, в том числе добавлена аутентификация, 
поэтому чтобы правильно запустить тесты, следует сначала в одном окне терминала запустить приложение 
с переменным окружения TODO_PASSWORD=12345:

```
TODO_PASSWORD=12345 go run .
```

Пароль обязательно должен быть 12345, так как в настройках тестов tests/setting.go используется токен с его использованием
Также при запуске тестов нужно передать в переменной окружении этот пароль:

```
TODO_PASSWORD=12345 go test ./tests
```

# Сборка и запуск проекта через docker
В репозитории имеется файл Dockerfile для сборки образа. Чтобы собрать образ нужно ввести команду:

шаблон для сборки образа: 	
```
docker build -t <image_name>:<tag> .
```
пример: 			
```
docker build -t final_project:v1 .
```

после сборки образа нужно запускать контейнер:

с применением переменных окружения: 
```
docker run -p 7540:7540 -e TODO_PASSWORD=12345 final_project:v1
```
